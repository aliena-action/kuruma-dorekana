# プライバシー・データマップ

「くるまどれかな?」感想送信バックエンドが扱うデータの一覧と、
保存しないデータの担保方法。

## 保存するデータ(D1 `feedback` テーブル)

| 列 | 内容 | 由来 | 個人情報該当性 |
|---|---|---|---|
| `id` | 連番 | D1 が自動採番 | なし |
| `independence` | ひとりで遊べたか(3択の内部値) | フォーム選択 | なし |
| `replay_interest` | また遊びたそうか(3択の内部値) | フォーム選択 | なし |
| `age_group` | 年齢区分(3択の内部値、未回答は NULL) | フォーム選択 | なし(区分のみで個人を特定しない) |
| `help_areas` | 手伝った部分(許可値のJSON配列文字列、未回答は NULL) | フォーム複数選択 | なし |
| `comment` | 自由記入(最大300文字、未回答は NULL) | フォーム入力 | **記入内容次第**(下記参照) |
| `submitted_at` | 送信日時(UTC, ISO 8601) | D1 の列デフォルトで自動設定 | なし |

## 保存しないデータと担保方法

| データ | 担保方法 |
|---|---|
| 名前・メールアドレス・住所・電話番号・カード情報 | 受理フィールドは5つのみの許可リスト方式。それ以外のキーを含むリクエストは保存せず 400 で拒否する |
| 任意の追加フィールド | 同上(未知フィールドは例外なく 400) |
| ブラウザの User-Agent | コードがリクエストヘッダーの User-Agent を一切読み取らない。保存先の列も存在しない |
| IP アドレス | コードが IP(`CF-Connecting-IP` 等)を一切読み取らない。保存先の列も存在しない |
| クライアント申告の送信日時 | `submitted_at` はクライアントから受け取らず(送信されたら 400)、D1 側で自動設定 |

テーブルに列が存在しないため、コード変更なしにこれらが保存されることはない。
新しい列やフィールドを追加する変更は、この文書の更新とセットで行うこと。

## 自由記入(`comment`)の注意

保護者が自由記入欄に自ら個人情報(名前・連絡先等)を書き込む可能性は排除できない。
軽減策:

- 最大300文字に制限(API側で強制)
- フォーム側に「個人情報(お名前・ご連絡先など)は記入しないでください」の注意書きを
  表示することを推奨(INTEGRATION.md 参照)
- 定期的に内容を確認し、個人情報を含む行は削除する運用を推奨

## Cloudflare 基盤側のログについて

このキットが制御するのは D1 への保存内容のみ。Cloudflare のアクセスログや
WAF レート制限は Cloudflare 基盤側で IP 等を一時的に処理するが、
このキットのコードが D1 へ書き込むことはない。

## 保持・削除

- 保持期間の要件は未定義。集計後に古い行を削除する運用を推奨
- 削除例: `DELETE FROM feedback WHERE submitted_at < '2026-01-01';`
